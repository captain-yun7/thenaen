# 더나은호흡 - 카페24 배포 및 Flutter APK 작업계획서

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 더나은호흡 (breathing) |
| 기반 프레임워크 | GnuBoard 5.5.13 + YoungCart 5.4.5.5.1 |
| 현재 도메인 | `breathing.webadsky.net` |
| 현재 DB 호스트 | `breathing.webadsky.net:3306` |
| 테마 | `basic` (데스크탑 + 모바일 반응형) |

## 고객 요구사항

1. 카페24에 호스팅하여 구동
2. 결제 기능 불필요 → 비활성화
3. 소셜로그인은 네이버/구글만 유지
4. Flutter WebView로 감싸서 APK 생성

## Phase 1: 코드 정리 (배포 전 준비)

### 1-1. 결제 기능 비활성화

**변경 파일:**
- `www/data/dbconfig.php` → `G5_USE_SHOP`을 `false`로 변경
- `www/theme/basic/theme.config.php` → 쇼핑몰 관련 메뉴/스킨 비활성화
- `www/extend/shop.extend.php` → 필요시 비활성화

**주의:** 상품 목록이나 카테고리 페이지가 사용되고 있을 수 있으므로, 단순히 `G5_USE_SHOP = false`로 변경 후 사이트 정상 작동 확인 필요

### 1-2. 소셜로그인 정리 (네이버/구글만 유지)

**현재 상태:** HybridAuth 기반으로 Facebook, Twitter, Kakao, Payco, Naver, Google 등 설정 가능
**변경 대상:**
- `www/extend/social_login.extend.php` → 네이버/구글 외 비활성화
- `www/plugin/social/config.php` → 네이버/구글만 enabled
- 관리자 패널에서 네이버/구글 OAuth 키 입력 필요

**고객에게 전달할 사항:**
- 네이버 개발자센터에서 OAuth 앱 등록 → Client ID / Secret 발급
- Google Cloud Console에서 OAuth 2.0 클라이언트 생성 → Client ID / Secret 발급
- 콜백 URL: `https://{카페24도메인}/plugin/social/`

### 1-3. FCM Legacy API → v1 마이그레이션

**현재 문제:** `www/bbs/cron_alrm.php`에서 FCM Legacy HTTP API(`fcm.googleapis.com/fcm/send`) 사용 중 → **2024년 6월 폐기됨**

**변경 작업:**
1. Firebase 프로젝트 생성 (또는 기존 프로젝트 확인)
2. 서비스 계정 JSON 키 발급
3. `cron_alrm.php`의 `PushSend()` 함수를 FCM v1 API로 교체
   - 엔드포인트: `https://fcm.googleapis.com/v1/projects/{PROJECT_ID}/messages:send`
   - 인증: OAuth 2.0 Bearer Token (서비스 계정 기반)
4. DB 접속 정보 중복 하드코딩 제거 → `_common.php` include 방식으로 변경

### 1-4. 하드코딩된 설정값 정리

| 파일 | 변경 내용 |
|------|-----------|
| `www/data/dbconfig.php` | 카페24 DB 정보로 교체 |
| `www/bbs/cron_alrm.php` | DB 직접 연결 → `_common.php` include, FCM 키 교체 |
| `www/config.php` | `G5_DOMAIN`, `G5_HTTPS_DOMAIN` 카페24 도메인으로 설정 |
| `www/config.php` | SMTP 계정 확인 (현재 Gmail 유지 또는 변경) |
| `www/plugin/social/config.php` | `base_url`을 `localhost` → 실제 도메인으로 변경 |

## Phase 2: 카페24 배포

### 2-1. 카페24 호스팅 설정

1. 카페24 웹호스팅 신청 (PHP + MySQL 지원 플랜)
2. FTP/SFTP 접속 정보 수령
3. PHP 버전 확인 (GnuBoard 5.5.x는 PHP 7.x ~ 8.1 지원)

### 2-2. DB 마이그레이션

1. `thegood.sql` 파일을 카페24 phpMyAdmin에서 import
2. `dbconfig.php`에 카페24 DB 접속 정보 입력
3. `g5_config` 테이블에서 도메인/경로 관련 값 수정
4. `cron_alrm.php`의 DB 접속 정보도 수정 (또는 `_common.php` include 방식으로 전환)

### 2-3. 파일 업로드

1. `www/` 디렉토리 전체를 카페24 웹 루트에 업로드
2. `data/` 디렉토리 퍼미션 확인 (707 또는 757)
3. `data/session/` 디렉토리 생성 및 퍼미션 설정

### 2-4. 크론잡 설정

카페24에서 크론잡 등록:
```
* * * * * /usr/bin/php /home/hosting_아이디/www/bbs/cron_alrm.php
```
- 매분 실행하여 복약/운동/병원 알림 처리

### 2-5. SSL 설정

1. 카페24 무료 SSL 인증서 발급
2. `config.php`에서 `G5_HTTPS_DOMAIN` 설정
3. HTTP → HTTPS 리다이렉트 `.htaccess` 설정

## Phase 3: Flutter WebView APK

### 3-1. Flutter 프로젝트 구조

```
thenaen_app/
├── android/
├── lib/
│   ├── main.dart              # 앱 진입점
│   ├── webview_screen.dart    # WebView 화면
│   └── firebase_service.dart  # FCM 토큰 관리
├── pubspec.yaml
└── android/app/google-services.json  # Firebase 설정
```

### 3-2. 필요 패키지

```yaml
dependencies:
  flutter:
    sdk: flutter
  webview_flutter: ^4.x
  firebase_core: ^3.x
  firebase_messaging: ^15.x
  url_launcher: ^6.x
```

### 3-3. 핵심 기능

1. **WebView**: 카페24 도메인 로드, 외부 링크 처리
2. **FCM 토큰 발급**: 앱 시작 시 FCM 토큰 생성
3. **토큰 전달**: JavaScript Bridge 또는 API를 통해 서버에 토큰 + device 타입 전달
4. **푸시 수신**: 포그라운드/백그라운드 푸시 알림 처리
5. **딥링크**: 푸시 클릭 시 해당 URL로 WebView 이동

### 3-4. 서버 측 토큰 저장 API

`g5_member` 테이블에 이미 `device`, `token` 필드가 존재하므로, 토큰 저장 API 엔드포인트 구현:
- `POST /bbs/fcm_token.php` → `mb_id`, `token`, `device` 저장

### 3-5. APK 빌드

```bash
flutter build apk --release
```
