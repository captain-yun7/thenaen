# 카페24 배포 가이드

> 더나은호흡 프로젝트를 카페24 웹호스팅에 배포하는 단계별 매뉴얼

---

## 1. 사전 준비

### 1-1. 카페24 호스팅 신청

- **권장 상품:** 웹호스팅 10G 이상 (PHP + MySQL 지원)
- **필수 조건:**
  - PHP 7.4 ~ 8.1 지원
  - MySQL 5.7 이상
  - 크론잡(Cron) 설정 가능
  - SSL 인증서 발급 가능

### 1-2. 호스팅 정보 확인

카페24 호스팅 관리 페이지에서 아래 정보를 확인합니다.

| 항목 | 예시 | 메모 |
|------|------|------|
| FTP 호스트 | `ftp.도메인.cafe24.com` | |
| FTP 계정 | `hosting_id` | |
| FTP 비밀번호 | `****` | |
| DB 호스트 | `hosting_id.cafe24.com` | |
| DB 이름 | `hosting_id` | |
| DB 계정 | `hosting_id` | |
| DB 비밀번호 | `****` | |
| 웹 루트 경로 | `/home/hosting_id/www/` | |

## 2. 데이터베이스 마이그레이션

### 2-1. SQL 파일 임포트

1. 카페24 관리 페이지 → **phpMyAdmin** 접속
2. 좌측에서 데이터베이스 선택
3. 상단 **가져오기** 탭 클릭
4. `thegood.sql` 파일 업로드 → 실행

> 파일 크기 제한이 있을 경우 phpMyAdmin의 업로드 제한을 확인하거나,
> SSH가 가능하다면 `mysql -u 계정 -p DB명 < thegood.sql` 명령어로 임포트

### 2-2. DB 데이터 수정

phpMyAdmin에서 아래 SQL을 실행하여 도메인 관련 설정을 업데이트합니다.

```sql
-- 관리자 설정에서 도메인 변경 (기존 도메인 → 새 도메인)
UPDATE g5_config SET cf_admin_email = '운영자이메일@도메인.com' WHERE cf_id = 1;

-- 소셜로그인: 네이버/구글만 활성화
-- 관리자 패널 → 기본환경설정 → 소셜로그인에서 설정 가능
```

## 3. 파일 업로드

### 3-1. FTP 업로드

1. FileZilla 등 FTP 클라이언트로 카페24 서버에 접속
2. `www/` 폴더의 **전체 내용**을 카페24 웹 루트(`/home/hosting_id/www/`)에 업로드
3. 업로드할 폴더/파일 목록:
   ```
   www/
   ├── adm/
   ├── bbs/
   ├── css/
   ├── data/
   ├── extend/
   ├── img/
   ├── js/
   ├── lib/
   ├── mobile/
   ├── plugin/
   ├── skin/
   ├── theme/
   ├── config.php
   ├── _common.php
   ├── index.php
   └── ... (기타 파일)
   ```

### 3-2. 퍼미션 설정

FTP 클라이언트에서 아래 디렉토리의 퍼미션을 변경합니다.

| 디렉토리 | 퍼미션 | 설명 |
|----------|--------|------|
| `data/` | `707` 또는 `757` | 파일 업로드, 세션, 캐시 저장 |
| `data/session/` | `707` | 세션 파일 저장 |
| `data/file/` | `707` | 첨부파일 저장 |
| `data/editor/` | `707` | 에디터 이미지 저장 |
| `data/tmp/` | `707` | 임시 파일 |
| `data/cache/` | `707` | 캐시 파일 |
| `data/firebase/` | `700` | Firebase 키 (웹 접근 차단됨) |

```bash
# SSH 접속이 가능한 경우
chmod 707 data data/session data/file data/editor data/tmp data/cache
chmod 700 data/firebase
```

## 4. 설정 파일 수정

### 4-1. DB 접속 정보 (`data/dbconfig.php`)

```php
define('G5_MYSQL_HOST', '카페24_DB_호스트');
define('G5_MYSQL_USER', '카페24_DB_계정');
define('G5_MYSQL_PASSWORD', '카페24_DB_비밀번호');
define('G5_MYSQL_DB', '카페24_DB_이름');
```

### 4-2. 도메인 설정 (`config.php`)

```php
// 26~27번째 줄
define('G5_DOMAIN', 'https://www.도메인.com');
define('G5_HTTPS_DOMAIN', 'https://www.도메인.com');
```

### 4-3. SMTP 설정 (`config.php`)

현재 Gmail SMTP를 사용 중입니다. 변경이 필요하면:

```php
// 182~189번째 줄
define('G5_SMTP', 'smtp.gmail.com');
define('G5_SMTP_PORT', '465');
define('G5_SMTP_SECURE', 'ssl');
define('G5_SMTP_USER', '이메일@gmail.com');
define('G5_SMTP_PW', '앱_비밀번호_16자리');
```

> Gmail 앱 비밀번호 발급: Google 계정 → 보안 → 2단계 인증 → 앱 비밀번호

### 4-4. FCM 설정 (`bbs/cron_alrm.php`)

```php
// 25번째 줄
define('FCM_PROJECT_ID', 'firebase-프로젝트-id');
```

### 4-5. Firebase 서비스 계정 키 배치

Firebase Console에서 다운로드한 `service-account.json` 파일을:

```
data/firebase/service-account.json
```

경로에 업로드합니다. `.htaccess`로 웹 접근이 차단되어 있으므로 보안상 안전합니다.

## 5. SSL 인증서 설정

1. 카페24 관리 페이지 → **보안서버(SSL)** 메뉴
2. **무료 SSL 인증서** 발급 신청
3. 발급 완료 후 `config.php`의 `G5_HTTPS_DOMAIN` 확인
4. HTTP → HTTPS 리다이렉트를 위해 웹 루트의 `.htaccess`에 추가:

```apache
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
```

## 6. 크론잡 설정

카페24 관리 페이지 → **크론잡** 메뉴에서 등록합니다.

| 항목 | 설정값 |
|------|--------|
| 실행 주기 | 매분 (`* * * * *`) |
| 실행 명령어 | `/usr/bin/php /home/hosting_id/www/bbs/cron_alrm.php` |

> 카페24에서 분 단위 크론잡을 지원하지 않는 경우, 5분 또는 10분 단위로 설정합니다.
> 이 경우 복약 알림의 정확도가 떨어질 수 있습니다.

## 7. 소셜로그인 설정

### 7-1. 네이버 로그인

1. [네이버 개발자센터](https://developers.naver.com/) 접속
2. **애플리케이션 등록** → 사용 API: "네아로(네이버 로그인)"
3. 서비스 URL: `https://www.도메인.com`
4. Callback URL: `https://www.도메인.com/plugin/social/`
5. 발급받은 **Client ID**와 **Client Secret**을 관리자 패널에 입력

### 7-2. 구글 로그인

1. [Google Cloud Console](https://console.cloud.google.com/) 접속
2. **API 및 서비스** → **사용자 인증 정보** → **OAuth 2.0 클라이언트 ID 만들기**
3. 애플리케이션 유형: 웹 애플리케이션
4. 승인된 리디렉션 URI: `https://www.도메인.com/plugin/social/?hauth.done=Google`
5. 발급받은 **Client ID**와 **Client Secret**을 관리자 패널에 입력

### 7-3. 관리자 패널에서 설정

1. `https://www.도메인.com/adm/` 접속 (최고관리자 계정)
2. **기본환경설정** → **소셜로그인** 탭
3. 네이버/구글의 키 입력 및 사용 체크

## 9. 트러블슈팅

### DB 연결 오류
- `dbconfig.php`의 호스트/계정/비밀번호 재확인
- 카페24에서 외부 DB 접근이 차단되어 있는지 확인

### 파일 업로드 안 됨
- `data/` 디렉토리 퍼미션 확인 (707)
- PHP `upload_max_filesize` 설정 확인

### 소셜로그인 오류
- Callback URL이 정확한지 확인 (끝에 `/` 포함 여부)
- SSL 인증서가 적용되었는지 확인 (HTTPS 필수)

### 크론잡 실행 안 됨
- PHP 실행 경로 확인: `which php` → `/usr/bin/php` 또는 `/usr/local/bin/php`
- 크론잡 로그 확인: `cron_alrm.php`의 error_log 출력 확인
