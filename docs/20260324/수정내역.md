## DB 확인 필요 항목

### 이슈 1: 이름이 아랍어처럼 깨져 보임 (ID/PW 로그인 시)

**증상:** ID/PW 로그인 시 "❓❓❓ξ❓❓❓님"처럼 이름 깨짐. 네이버 로그인에서는 정상.

**분석 결과:**
- 코드 자체에는 문제 없음 (UTF-8 charset 설정, DB charset utf8 모두 정상)
- 해당 사용자의 `g5_member` 테이블 `mb_name` 필드에 잘못된 인코딩으로 저장된 것으로 추정
- 네이버 로그인 사용자와 ID/PW 사용자는 다른 계정(다른 mb_id)이므로 DB 데이터 자체가 다름

**해결 방법:**
1. DB에 접속하여 해당 사용자의 `mb_name` 값 확인:
   ```sql
   SELECT mb_id, mb_name, mb_nick, HEX(mb_name) FROM g5_member WHERE mb_id = '해당ID';
   ```
2. 이름이 깨져있다면 올바른 값으로 UPDATE:
   ```sql
   UPDATE g5_member SET mb_name = '올바른이름' WHERE mb_id = '해당ID';
   ```

### 이슈 4: 월간수행기록 그래프 안 보임 (ID/PW 로그인 시)

**증상:** ID/PW 로그인 시 월간수행기록 페이지에서 날짜/그래프가 완전히 비어있음. 네이버 로그인에서는 정상.

**분석 결과:**
- 이슈 1과 연관될 가능성 높음 (같은 사용자 계정의 데이터 문제)
- `my.php`의 달력 렌더링 코드는 사용자 데이터와 무관하게 동작해야 하지만, PHP 오류로 렌더링이 중단되고 있을 가능성
- 해당 사용자의 `mb_name`에 잘못된 바이트가 있으면 SQL 쿼리 등에서 PHP 오류 발생 가능

**해결 방법:**
1. 이슈 1의 DB 수정을 먼저 진행
2. 수정 후에도 문제가 지속되면 PHP 에러로그 확인 필요
